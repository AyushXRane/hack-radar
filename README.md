# hack-radar

hack-radar is a hackathon project agent skill that monitors a team’s progress and helps prepare for judging, competition, and final submissions.

It analyzes git commit velocity, sponsor API usage, project drift from the original idea, competing Devpost submissions in the same prize track, alignment with judging criteria, and final submission completeness. It can also generate demo scripts and slide outlines directly from the codebase and commit history.

Built for hackathon teams with the following concerns:

- Are we building fast enough?  
- Are we actually using sponsor APIs properly?  
- Have we drifted too far from the original idea?  
- Are competing teams building something similar?  
- How well do we score against the judging rubric?  
- Is our Devpost submission complete?  
- Are we ready to demo and pitch?
